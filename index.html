DISCORD BOT
npbot
Dates

December 2017 - ?

Technologies

  C

Description

This is a bot providing the opportunity to tell your Discord friends what song you are listening to in the video game called osu!.

In-game, typing the command "/np" in the chat sends a message with the song your are currently listening to.

We have a server observing the chat messages sent to the bot account, using an irc client. If a message corresponds to the info produced by the "/np" command, a JSON is built with the infos, and sent to the server running the Discord bot. The bot sends a message in the chat with the data.

People involved

This project involves 3 people. I was in charge of programming the code ran on our server for the chat processing, and I decided what irc client to use, and configured it.

Unity3D / C#

Description

During the class about Video Game conception, given at the Université du Québec à Chicoutimi (UQAC), I took part in a 4 member team, and we designed a game prototype and developed it in 4 weeks.

MASSharing
Dates

July 2017 - ?

Technologies

HTML / CSS / JavaScript /   PHP

Description

MASSharing stands for Movies, Animes, Series sharing.

This website is a video library, collecting infos from databases. I started working on this after a discussion with friends, when we suggested that one of us should host a server to store all of our video files, so that we can share all of libraries.

I decided to work on this because I thought it would be better to have our own interface for the server, and that it would be a good experience.

The project is still a WIP and currently in standby as I focus on other projects. Only the Anime side, using MyAnimeList database, has been developped.

In its current state, it is possible to log in, to choose the section, to see the different files (sorted by anime), to see the details about an anime and the list of episodes on the server, and to watch or download them. The administrator also have the possibility to upload files, and to refresh the local database.

VIRTUAL REALITY APPLICATION
Urbawood VR
Dates

January 2017 - April 2017

Technologies

Unity3D / C#

Context

This project was organized by my school, ISEN. They hold an appartement, showcasing a modern, connected and smart environment.

We were asked to work on a virtual reality application that would simulate the appartement, so that they could do demonstrations anywhere.

Description

A previous project involved the realization of a mobile application to show the appartement, based on the virtual architect plans.

Our first goal was to include the HTC Vive headsets controls and display to the existing project. Then, we had to improve the visual quality, as the mobile version had poor graphics settings, but our computer had a Nvidia GTX1080 GPU, with a lot more ressources. We tweaked the lighting settings, and found better 3D models. We also added more objects, so that the scene would feel alive.

We also worked on making the appartment interractable. Some objects could be moved, but also, the television could be controlled, and this was directly linked to the real television.

As a bonus, I also got the opportunity to work with the Kinect from Microsoft, and worked on a way to display the real world into the appartment. That way, we could synchronise the two environments (when using the headset inside the appartment) and it was possible to feel the objects displayed in the virtual world. Also, you could see your body, and the people around you.

People involved

We were 5 to work on this. I searched for some of the 3D models, but I mostly worked on the lighting/performance optimization. The models we added were designed for "photo-like rendering" and i worked on reducing their complexity. I also redesigned some elements using the original plans (they contained objects too).

I was also in charge of the configuration part of the program, so that it was possible to move the environnement from inside the game, to make it coincide with the real environment. Finally, as mentionned above, I worked with the Kinect, and connected it to our project (you can see how the real world was represented in one of the pictures).

RESEARCH PROJECT
CMPprojetM1
Dates

October 2016 - December 2016

Technologies

Python

Context

This school project was proposed by two teachers, interested in the performances a GPU could offer for massive data computing, especially for noise maps, represented by huge matrix with a lot of empty values.

Description

For the project, we were four. The project was organized in different steps. The first step was about doing a comparison between the available technologies, both for CPU and GPU. We did this with matrix multiplication. We split the group in 2 pairs, and I worked on the GPU implementation, using pyOpenCL.

After this, each duo could implement a more complexe algorithm of its choice, to push the comparison further. We decided to work on cracking SHA256-hashed password.

All of our reports can be found on the repository (in French).

Context

This 3 weeks school project is based on the Constellation platform. This home automation tool gives the opportunity to easily link a wide range of different devices together.

The application uses the ionic developement framework, and we wrote different packages for the Constellation environment.

I also supervised the migration from our first test website to the ionic application, and worked on its design. And I implemented the webcam snapshot function. When using this, a picture taken from the computer webcam is sent to the phone running the application through PushBullet.
